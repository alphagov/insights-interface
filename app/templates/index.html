Insights Reporting Interface

Demonstration of a new potential interface through which Insights reporting content can be served.

The interface utilises a series of technical tools to enable the curation (MarkDown), organisation (Flask, Python) and hosting (Google Cloud Run) of Insights reporting content in a browser-accessible interface.

The demo utilises the GOV.UK design system to utilise the familiar look, feel, and accessibility features of GOV.UK.

Feedback sentiment over time

{% include 'sentiment_over_time_2022-10-11.html' %}

Feedback analysis is performed to create a depiction of user feedback sentiment over time.

Multiple sentiment analysis models are used to create separate sentiment predictions for each sentence in feedback records for a given period.

Estimated sentiment is then averaged across all models used to create an 'overall' sentiment prediction.

Feedback (bigram) topics over time

{% include 'GSDMM_topics_over_time_2022-10-11.html' %}

Feedback records are analysed using a 'Movie Group Process' which estimates probability of topic similarity between feedback sentences based on frequently used terms.

A number of 'passes' of this analysis are performed to identify the most coherent 'groups' (topics) and to identify which feedback sentences belong to each group.

These groups can be reprented by 'bigram' terms frequently observed in each topic.

Each bigram has an estimated 'probability' of association to a given topic; outputs bigrams are arranged in descending order of probability.

Feedback topics over time

{% include 'BERT_topics_over_time_2022-10-11.html' %}

The 'BERT' topic model is used to identify common terms within whole feedback records.

This process works in a similar way to the GSDMM bigram analysis, though topics are referenced according to single associated words.

These groups can be reprented by terms frequently observed in each topic.
